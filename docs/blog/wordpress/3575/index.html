Шаблоны для всех типов записей в версии 4.7

  • Автор записи Автор: Юлия Широкова
  • Дата записи 19.02.2017
  • Комментарии

WordPress вот уже 12 лет поддерживает пользовательские шаблоны страниц, позволяя разработчикам создавать разнообразные разметки для определённых страниц. И хотя эта функция является очень полезной, она всегда ограничивалась страницами и не была доступна для других типов записей. WordPress 4.7 меняет это.

Открывая функционал шаблона страницы для всех типов записей, гибкость структуры шаблона продолжает усовершенствоваться.

В дополнение к заголовку файла Template Name, типы записей, поддерживаемые шаблоном, могут быть установлены с помощью Template Post Type: post, foo, bar. Например:

<?php
/*
Template Name: Full-width layout
Template Post Type: post, page, product
*/
// … your code here

Таким образом у вас будет возможность выбрать этот full-width шаблон для записей, страниц и товаров.

Когда существует как минимум один шаблон для типа записи, блок «Параметры Записи» («Post Attributes») будет отображаться в бэк-энде (админке) без необходимости добавления поддержки типа записей для ‘page-attributes’ или чего-либо ещё. Надпись «Параметры Записи» («Post Attributes») может быть настроена для каждого отдельного типа записи с помощью метки ‘attributes’ во время регистрации типа записи.

 

Совместимость с предыдущими версиями

Допустим вы хотите опубликовать тему с поддержкой шаблонов для типов записей. Версии WordPress ниже 4.7 будут игнорировать заголовок Template Post Type и показывать шаблон в списке шаблонов страницы, даже если он работает только для обычных записей. Чтобы это предотвратить, вы можете подключиться к фильтру theme_page_templates, чтобы исключить его из списка . Пример:

/**
* Hides the custom post template for pages on WordPress 4.6 and older
*
* @param array $post_templates Array of page templates. Keys are filenames, values are translated names.
* @return array Filtered array of page templates.
*/
function makewp_exclude_page_templates( $post_templates ) {
    if ( version_compare( $GLOBALS['wp_version'], '4.7', '<' ) ) {
        unset( $post_templates['templates/my-full-width-post-template.php'] );
    }
    return $post_templates;
}
add_filter( 'theme_page_templates', 'makewp_exclude_page_templates' );

Таким образом вы можете поддерживать пользовательские шаблоны типов записей в WordPress 4.7 и ниже, сохраняя полную совместимость с предыдущими версиями.

Обратите внимание, что theme_page_templates на самом деле динамический фильтр theme_{$post_type}_templates. Динамическая доля хука — $post_type —  ссылается на тип записи, поддерживаемый шаблонами. Например, вы можете подключится к theme_product_templates, чтобы отфильтровать список шаблонов для типа записи product.
